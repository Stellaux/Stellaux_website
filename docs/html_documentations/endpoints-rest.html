Health & metrics

Operational endpoints used by the load balancer and the monitoring stack. Not for client use.

GET /healthz none
Purpose

Liveness probe. Returns 200 if the process is up. No DB call.

Response 200
{ "status": "ok" }
GET /readyz none
Purpose

Readiness probe. Pings the database; returns 503 if Postgres is unreachable or the pool is exhausted. The LB routes traffic only when this returns 200.

Response 200
{ "status": "ready", "db": "ok" }
Response 503
{ "status": "not_ready", "db": "unreachable" }
GET /metrics none
Purpose

Prometheus exposition format. In production, this route is firewalled to the monitoring network — not exposed publicly.

Sample
# HELP stellaux_orders_total Total orders placed.
# TYPE stellaux_orders_total counter
stellaux_orders_total{source="website"} 142
http_requests_total{method="GET",endpoint="/api/v1/catalog/products",status="200"} 8401

Full metric inventory in Observability → Metrics.

Craft builder public

Drives the per-component compatibility UI for the build-your-own product. Reads products filtered by craft_role plus the craft_compatibility join table.

GET /api/v1/craft/bases none
Description

List products with craft_role = 'base'. Includes craft_base_type for grouping (pendant / chain / trunk).

GET /api/v1/craft/accessories none
Description

List products with craft_role = 'accessory'.

GET /api/v1/craft/compatibility/{base_handle} none
Description

Return the list of accessory product ids compatible with a given base. The client uses this to gate accessory selection.

Response 200
{ "base_handle": "trunk-base", "accessory_ids": ["uuid-a","uuid-b"] }
Implementation

Single SELECT accessory_product_id FROM craft_compatibility WHERE base_product_id = (SELECT id FROM products WHERE handle = $1). Cached aggressively (s-maxage=300) — admin compatibility edits purge the cache via tag.

Authentication public

First-party email + password auth. Issues HS256 JWTs signed with JWT_SECRET, TTL JWT_EXPIRY_SECONDS. Supabase JWTs are also accepted on protected routes when SUPABASE_JWKS_URL is configured — see External Services.

POST /api/v1/auth/signup none
Request body
FieldTypeNotes
emailstring reqLowercased server-side.
passwordstring reqMin 12 chars; hashed with Argon2id.
namestringDisplay name.
Response 201
{ "user": { "id": "uuid", "email": "…" }, "access_token": "ey…", "expires_in": 3600 }
Errors

  • 422 WEAK_PASSWORD — validation failed.
Metrics

stellaux_auth_events_total{kind="signup",success="true|false"}

POST /api/v1/auth/login none
Request body
{ "email": "…", "password": "…" }
Response 200
{ "access_token": "ey…", "expires_in": 3600, "user": { "id": "uuid", "email": "…" } }
Failure
  • 401 INVALID_CREDENTIALS — constant-time compare; same error for unknown email or bad password to avoid enumeration.

Per-IP token bucket: 10 attempts / 5 min. Adopt Redis-backed limiter at S3 (see Scaling).

POST /api/v1/auth/forgot-password none
Request body
{ "email": "…" }

Always returns 200, regardless of whether the email exists, to prevent account enumeration. Sends a Resend email with a single-use token (TTL 30 min) when the account exists.

Webhooks verified

Public routes whose handlers verify HMAC signatures over the raw request body before any JSON parse. Idempotent on (source, external_id) via the webhook_events table — replays are safe.

POST /api/v1/webhooks/stripe HMAC sig
Headers
Stripe-Signature reqHMAC over raw body using STRIPE_WEBHOOK_SECRET.
Content-Typeapplication/json
Body limit

Webhook routes use the larger webhook_body_limit_bytes (default ~1 MB; Stripe payloads stay well under this).

Events handled
EventAction
checkout.session.completedInsert orders + order_items; decrement inventory_levels.on_hand; trigger Shippo label purchase; queue confirmation email.
checkout.session.expiredRelease reserved inventory; mark cart abandoned.
payment_intent.payment_failedRelease reservations; log; no email.
charge.refundedSet orders.fulfillment_status = 'refunded'; restock if pre-fulfillment; email customer.
charge.dispute.createdAdmin alert email; no auto-refund.
Responses
  • 200 — accepted (also returned for already-processed events).
  • 400 INVALID_SIGNATURE — body fails HMAC check.
  • 500 — handler error; Stripe retries up to 3 days.
Implementation
  1. Read raw body with request.into_body().collect().
  2. Verify signature in constant time.
  3. INSERT INTO webhook_events(source='stripe', external_id=event.id, payload=…) ON CONFLICT DO NOTHING.
  4. Switch on event type; commit DB transaction.
  5. Update webhook_events.processed_at.
POST /api/v1/webhooks/shippo HMAC sig
Headers
X-Shippo-Signature reqHMAC-SHA256 with SHIPPO_WEBHOOK_SECRET over raw body.
Events handled

track_updated — maps Shippo tracking_status.status to orders.status + updates tracking_* columns. On DELIVERED, queues a follow-up email scheduled for +3 days.

Cart protected

JWT required. Anonymous carts identified by signed cookie cart_token are supported via the protected fallback path (the extractor accepts either a Bearer token or a verified anon-token cookie).

GET /api/v1/cart JWT or anon cookie
Description

Return the current cart with line items grouped by assembly_id (for Craft bundles). Lazily creates an empty cart on first call.

Response 200
{
  "cart_id": "uuid",
  "currency": "USD",
  "items": [
    { "id": "uuid", "variant_id": "uuid", "sku": "HAL-PEND-S",
      "qty": 1, "unit_price_cents": 12800,
      "assembly_id": null, "assembly_role": null,
      "product_name": "Halo Pendant", "image_url": "…" }
  ],
  "subtotal_cents": 12800
}
POST /api/v1/cart/items JWT or anon
Request body
FieldTypeNotes
variant_iduuid reqFK to product_variants.
qtyint req> 0.
assembly_iduuidGroups Craft components in one purchase.
assembly_roleenumbase | accessory
Behavior
  • Validates available > 0 for the variant.
  • Snapshots unit_price_cents from product_variants.price_cents at insert time.
  • If a line for the same (cart_id, variant_id, assembly_id) exists, increments its qty.
Errors
  • 409 INVENTORY_UNAVAILABLE — variant out of stock.
  • 404 VARIANT_NOT_FOUND
PATCH /api/v1/cart/items/{item_id} JWT or anon
Request body
{ "qty": 2 }

qty must be > 0; for removal, use DELETE.

DELETE /api/v1/cart/items/{item_id} JWT or anon

Remove a single line item. If the item is part of an assembly, also removes the sibling lines with the same assembly_id.

DELETE /api/v1/cart JWT or anon

Empty the cart (deletes all line items, keeps the cart row alive).

Checkout protected

Two-step flow: rate quote, then session creation. Inventory is reserved inside the session-creation transaction; reservations expire automatically on Stripe's checkout.session.expired webhook.

POST /api/v1/checkout/shipping-rates JWT
Description

Compute total weight + dims from the cart, call Shippo POST /shipments, and return 2–3 service-level rates.

Request body
{
  "cart_id": "uuid",
  "address": {
    "recipient": "Y. Aliyev",
    "street": "…", "city": "…", "state": "CA",
    "postal_code": "94110", "country": "US",
    "phone": "+1-…"
  }
}
Response 200
{
  "rates": [
    { "shippo_rate_id": "…", "carrier": "USPS", "service": "Priority",
      "amount_cents": 850, "estimated_days": 3 }
  ]
}
Errors
  • 400 ADDRESS_INVALID — Shippo flagged the address as undeliverable.
  • 503 SHIPPO_UNAVAILABLE — upstream timeout; client falls back to flat-rate.
POST /api/v1/checkout/session JWT
Description

Reserve inventory, create a Stripe Checkout Session, return the hosted URL. The browser redirects there.

Request body
{ "cart_id": "uuid", "shippo_rate_id": "…", "address": { … } }
Response 200
{ "checkout_url": "https://checkout.stripe.com/c/pay/cs_test_…",
  "session_id":   "cs_test_…",
  "expires_at":   "2026-05-29T14:30:00Z" }
Implementation
  1. Open Postgres tx; for each cart item run SELECT … FOR UPDATE on inventory_levels (advisory lock per variant_id for hot SKUs).
  2. UPDATE inventory_levels SET reserved = reserved + qty.
  3. Call Stripe REST POST /v1/checkout/sessions with line items + shipping option + metadata.
  4. Store stripe_checkout_session_id on the cart; commit.
Errors
  • 409 INVENTORY_UNAVAILABLE — a variant went out of stock between rate quote and session create.
  • 502 STRIPE_UNAVAILABLE — reservation rolled back.

POST/api/v1/admin/orders/{order_id}/canceladmin

Marks the order cancelled, releases reserved inventory if pre-fulfillment, voids the Shippo label if not yet handed to carrier.

Reprint or repurchase a shipping label. Stores the new PDF in object storage at shipping-labels/{order_id}-{ts}.pdf.
